## 0. 정체성

**킨더버스(KinderVerse) 게임 뷰어** — 보드 툴바에 추가되는 모듈. 유치원 교사가
프롬프트로 지시하면, 교사+아이 함께 또는 아이 혼자 할 수 있는 **아주 심플하지만
터치감·반응·보상 퀄리티가 높은** 인터랙티브 게임이 생성·플레이된다.

킨더버스는 i-screamkids(시공그룹)의 AI 에이전트 기반 워크스페이스다. 이 뷰어는
킨더버스의 기존 구조(3-tier 에이전트 / React Flow 보드 / Workflow Lane / Milray Park
디자인 시스템) 위에 얹힌다.

## 1. 가장 중요한 설계 결정 (절대 흔들지 말 것)

### ❶ 런타임 코드 생성 ❌ → 템플릿 + 파라메트릭 콘텐츠 ✅

게임을 만드는 방식은 **"잘 만든 게임 엔진(템플릿)을 두고, AI는 콘텐츠 파라미터(GameSpec
JSON)만 생성"** 한다. 매번 코드를 새로 생성하지 **않는다**.

이유:
- **속도**: 흔한 케이스는 에셋/TTS가 캐시/CDN에서 즉시 → 1초 미만. 코드 생성은 수 초.
- **퀄리티 일정성**: 정제된 엔진에서만 터치감·애니메이션·보상이 항상 일정하게 보장됨.
- **안전성**: 큐레이션된 에셋 + 고정 엔진이라 부적절 콘텐츠가 구조적으로 불가능.

> 아이가 느끼는 "와!"는 코드 다양성이 아니라 **반응하는 애니메이션·만족스러운 터치
> 피드백·화려한 보상·음성**에서 나온다. 이건 미리 만든 엔진에서만 나온다.

"그냥 프롬프트로 만들어줘"는 그대로 작동한다 — AI가 프롬프트를 **템플릿 선택 + 콘텐츠
생성**으로 매핑하기 때문. (예: "동물원 동물 세기 게임" → `counting` 템플릿 + {사자,
코끼리...} 콘텐츠)

### ❷ GameSpec이 단일 계약(Single Contract)이다

`src/game-viewer/schema/gameSpec.ts`. AI가 산출하고, 모든 템플릿이 소비한다.
에셋 source(openmoji/teacher/generated)는 이 계약이 추상화하므로 **템플릿은 에셋
출처를 몰라도 렌더링된다.**

### ❸ 교사 이미지 교체는 "새 기능"이 아니라 "새 source 타입"이다

교사가 올린 사진은 `GameAsset.source = "teacher"`로 들어온다. 업로드 → 배경제거 →
정규화 → (실루엣) → 안전검증 파이프라인이 이를 "엔진 품질 에셋"으로 정제한다.
**처음부터 source 추상화로 짰기 때문에** 나중에 끼우는 게 깔끔하다. (M1 스키마에 이미 반영됨)

### ❹ 입구는 셋, 출력은 하나(GameSpec) — 프롬프트가 유일한 시작점이 아니다

GameSpec이 단일 출력이라 **입구를 늘리는 건 구조적으로 거의 공짜**다. 엔진은 입구가 뭐였는지
전혀 모른다. 세 입구를 **모두 1급**으로 지원한다:

1. **템플릿 갤러리에서 시작** — 교사가 게임 유형(숫자 맞추기 등)을 먼저 고른다. **기본 화면.**
   유치원 현장은 "오늘 과일 세기 활동"처럼 활동 유형이 먼저 정해지는 경우가 많아, 빈
   프롬프트창보다 발견성·인지부담 면에서 압도적으로 낫다.
2. **프롬프트로 시작** — "동물원 동물 세기 게임 만들어줘". **빠른 길.**
3. **예시에서 변형** — 스타터 라이브러리의 기존 게임을 열어 소재만 교체. 교사 이미지 교체와 연결.

**템플릿 선택 후 콘텐츠 채우기 = 폼 우선 + 프롬프트 옵션 (확정 방식, "C"):**
- 템플릿을 고르면 **간단 폼**으로 기본을 완성한다. ("무엇을 셀까요? [과일][동물][탈것]",
  "몇 개까지?", 연령) — 탭 선택만으로 글 입력 없이 끝까지 간다.
- 원하면 그 폼 안에서 **자유 프롬프트로 미세조정**한다. ("우리 반 텃밭 채소들로")

> 🔑 **폼 경로는 LLM 호출 없이 동작한다.** 선택값 → `buildSpecFromForm()`이 GameSpec을 결정적으로
> 조립한다(보기 숫자는 산술로 생성, 아이템은 카테고리별 큐레이션 셋에서 OpenMoji ref로 해석).
> 그래서 프롬프트 경로보다 **더 빠르고 더 안정적**이다. 프롬프트/미세조정만 LLM(Router→전문 에이전트)을 탄다.

> 결과적으로 **엔진·템플릿 렌더러는 손대지 않는다.** 추가되는 건 입력 UI(`entry/`)와, 폼 선택을
> GameSpec으로 바꾸는 얇은 빌더(`generate/buildSpecFromForm.ts`)뿐이다.

## 2. 기술 스택 (확정 — 임의로 바꾸지 말 것)

| 용도 | 라이브러리 | 비고 |
|---|---|---|
| **반응형 벡터 캐릭터** | **Rive** (`@rive-app/react-canvas`) | 감정 공감 게임의 핵심. 상태머신으로 입력에 반응. 정적 이미지로 못 내는 퀄리티 |
| **전환/제스처/드래그/스프링** | **Motion** (구 framer-motion) | 모든 등장·터치 피드백의 "쫀득함" |
| **에셋 백본** | **OpenMoji** (SVG) | 동물·식물·탈것·음식·과일·표정·사물·직업 전부 커버. 실루엣 변환·리컬러 공짜. 안전성 본질 보장. **기본 경로** |
| **드래그/히트검출 캔버스** | **Konva** (`react-konva`) | 선잇기 등 무거운 드래그 게임 |
| **사운드** | **Howler.js** | 효과음·스프라이트 |
| **한국어 음성(TTS)** | **Naver CLOVA Voice** | 유아 친화 한국어. 모든 지시/피드백을 음성으로. (표현력 중요한 곳은 ElevenLabs 병행 가능) |
| **보상 연출** | **canvas-confetti** + **Lottie** (`lottie-react`) | 별·폭죽·캐릭터 환호 |
| **상태** | **Zustand** + **zundo** | 킨더버스 표준 |
| **교사 이미지 배경제거** | **@imgly/background-removal** | 브라우저 온디바이스. 아이 사진 프라이버시에 1순위. 폴백: remove.bg / Replicate(BiRefNet) |
| **스키마 검증** | **zod** | GameSpec 런타임 검증 |

### 의도적으로 빼는 것
- **Phaser / PixiJS 같은 풀 게임 엔진** ❌ — 무겁고 React/React Flow 통합이 번거롭다.
  "심플하지만 고퀄"엔 위 경량 조합이 정확히 맞다.
- **런타임 이미지 생성** ❌ (핫패스에서) — OpenMoji에 없는 희귀 소재만 **폴백**으로,
  스타일 락 + 안전 분류기 거쳐. 기본 경로가 아니라 예외 경로.

> ⚠️ 패키지 버전: package.json의 버전은 보수적으로 잡혀 있다. `npm install` 후 각
> 라이브러리의 **현재 최신 호환 버전과 API를 확인**하고 필요시 업그레이드할 것
> (특히 Rive / Motion은 API 변동 가능).

## 3. 3-tier 에이전트 매핑 (킨더버스 구조 그대로)

| Tier | 역할 | 게임 뷰어에서 |
|---|---|---|
| **Router** | 프롬프트 → 의도 | 프롬프트 → `templateId` + 상위 파라미터(테마/개수/연령) |
| **전문 에이전트** | 구조화 생성 | rounds(아이템·정답·오답 보기) JSON 생성 + 유아 기준 교육적 검증 |
| **결정형 도구** | 확정적 처리 | 에셋 리졸버(아이템→OpenMoji), TTS 도구(캐시), 안전 검증기 |

> 런타임 에셋/TTS 호출은 **MCP가 아니라 백엔드 직접 API**로 (지연·캐싱 통제).
> MCP는 *저작/에이전트 오케스트레이션* 레이어에서만 의미가 있다.

## 4. 하드 규칙 (Child Safety & 유아 적합성)

이 모듈은 **유아 대상**이다. 아래는 협상 불가:

1. **에셋은 OpenMoji-first.** 부적절 이미지가 구조적으로 불가능한 경로를 기본값으로.
2. **읽기 의존 0.** 모든 지시·피드백·라벨은 **음성(TTS)** 으로 제공. 텍스트는 교사/큰 아이 보조용.
3. **터치 타깃 크게.** 작은 손가락 기준. 최소 64×64 권장, 간격 충분히.
4. **긍정 피드백만.** 틀려도 벌점·부정 연출 없음. 다정한 재시도 유도. ("다시 해볼까요?")
5. **연령 보정.** `ageRange` 3-5 / 5-7로 난이도(보기 수·개수 범위) 조절.
6. **교사 업로드 이미지는 반드시 안전 분류기 통과.** 실패 시 `status: "rejected"` → 게임 진입 차단.
7. **아이 사진 처리는 온디바이스 우선.** 얼굴을 외부 API로 보내는 걸 최소화.
   아이 얼굴 **일러스트화/변형은 기본 비활성** (원본 보존 + 누끼 + 프레임까지만).
8. **기관 격리 저장.** 업로드 이미지는 Supabase Storage 기관별 버킷. 동의·보관 정책 준수.

## 5. 디자인 시스템 (게임 뷰어 전용 — Milray Park 미적용)

> 🔴 **게임 뷰어 내부는 Milray Park 디자인 시스템을 따르지 않는다.** 교사용
> 워크스페이스(차분한 coral/serif)와 아이가 직접 만지는 게임 화면은 목표가 다르다.
> 게임 화면은 **아이가 좋아할 파스텔·큐트** 톤으로 간다.
>
> 경계: 게임을 감싸는 **보드 노드 프레임·툴바·프롬프트 입력**(교사가 보는 부분)은
> Milray Park 유지. **게임 플레이 화면 안쪽**(아이가 보는 부분)만 파스텔 시스템 적용.

### 게임 뷰어 파스텔 토큰 (`src/game-viewer/theme.ts`로 생성)
부드럽고 둥글둥글하고 따뜻한 파스텔. 채도 낮고 명도 높게.

```
배경      cream/연하늘    #FFF9F2 / #EAF4FB  (라운드별 살짝 다른 파스텔 배경 허용)
1차 컬러  파스텔 코랄      #FFB5A7
2차 컬러  파스텔 민트      #B5EAD7
3차 컬러  파스텔 옐로우    #FFE9A8
4차 컬러  파스텔 라벤더    #C7CEEA
5차 컬러  파스텔 피치      #FFD6BA
정답 강조 민트/그린        #95E1A3
텍스트    소프트 차콜      #5A5A66  (순수 검정 금지 — 너무 강함)
```

- **폰트**: 둥글고 친근한 한글 서체 (예: Pretendard의 둥근 변형 / Gmarket Sans / 나눔스퀘어
  라운드 등 둥근맛 폰트). 화면 텍스트는 보조이고 음성이 주이지만, 보이는 글자는 큼직·둥글게.
- **형태**: 큰 둥근 모서리(border-radius 24px+), 말랑한 그림자, 외곽선은 두껍고 부드럽게.
- **모션**: 통통 튀는 스프링(bounce), 살짝 과장된 스케일. 말랑·귀여운 느낌.
- **보상 연출**: 파스텔 색종이(confetti), 반짝이 별, 캐릭터 환호 — 한 번에 몰아서
  (orchestrated). 흩뿌린 효과보다 정돈된 한 방이 더 세다. **"화려하지만 정제된"** 유지.
- **카피**: 아이 눈높이의 짧고 다정한 한국어, 능동태. ("맞았어요!", "다시 해볼까요?")

> confetti/별/캐릭터는 위 파스텔 팔레트에서만 색을 뽑는다. 원색 형광은 피한다(눈 피로).
> 귀엽되 정제되게 — 유치하거나 산만하지 않게.

## 6. 환경 / 컨벤션 (중요)

- **OS**: Windows. **터미널**: PowerShell.
- **🔴 경로 공백 주의**: 루트가 `D:\claude_project\kinderverse concept` — **"concept" 앞에
  공백**이 있다. 모든 터미널 명령에서 경로를 **반드시 따옴표로 감쌀 것.**
  ```powershell
  cd "D:\claude_project\kinderverse concept"
  npm run dev   # 스크립트 자체는 OK, 경로 인자만 조심
  ```
- **언어**: 코드/식별자/주석 키워드는 영어. UI 카피·문서 산문은 한국어.
- **타입**: TypeScript strict. GameSpec 계약을 우회하는 any 금지.
- **구조**: 게임 뷰어는 `src/game-viewer/` 아래 모듈로. 나중에 킨더버스 본 앱에 복사 가능하게 자기완결적으로.

## 7. 디렉토리 구조 (목표)

```

  schema/
    gameSpec.ts          # 단일 계약 (제공됨)
    examples.ts          # 픽스처 (제공됨)
    gameSpec.zod.ts      # zod 검증 (M1에서 생성)
  engine/
    GameViewer.tsx       # 진입점. spec 받아 템플릿 라우팅
    GameShell.tsx        # 공통 셸: 지시 음성, 라운드 진행, 보상 오케스트레이션
    useGameAudio.ts      # Howler + TTS 재생 훅
    rewards.tsx          # confetti/Lottie/캐릭터 환호
  templates/
    counting/            # M1
    silhouette/          # M1
    emotion/             # M2 (Rive)
    matching/            # M2 (Konva)
  assets/
    openmoji.ts          # 아이템 라벨 → OpenMoji ref 리졸버 + CDN
  generate/
```
