# PRD — 킨더버스 게임 뷰어

## 1. 비전

유치원 교사가 **프롬프트 한 줄**로, 교사+아이 함께 또는 아이 혼자 할 수 있는
**심플하지만 고퀄리티**의 인터랙티브 게임을 즉시 만들어 플레이한다. 보드 툴바에서
호출되어 Workflow Lane 위 아티팩트 노드로 등장한다.

성공 기준 3가지(절대):
- **빠르다** — 에셋 라이브러리 경로 1초 미만.
- **퀄리티가 항상 높다** — 터치감·반응·보상이 일정하게 우수.
- **아이에게 적합하다** — 안전·읽기무관·긍정 피드백.

---

## 2. 아키텍처 핵심: 템플릿 + 파라메트릭 콘텐츠

(상세 근거는 CLAUDE.md §1)

```
 ┌─ ① 템플릿 갤러리 (기본) ─ 유형 선택 → 간단 폼 ─┐
 │                              └─(옵션) 프롬프트 미세조정 ─┐
 ├─ ② 프롬프트 (빠른 길) ──────────────────────────────────┤
 ├─ ③ 예시에서 변형 (M3) ─ 기존 게임 소재만 교체 ──────────┤
 │                                                          ▼
 │   [폼 경로] buildSpecFromForm()  ── LLM 없음, 결정적 ──►  │
 │   [프롬프트 경로] Router → 생성 에이전트 → 결정형 도구 ──►│
 │       (templateId+파라미터 → rounds 구조화 → 에셋/TTS/안전)
 │                                                          ▼
 └───────────────────────────────────►  GameSpec (JSON)
                                              │
                                              ▼
                                    [템플릿 렌더러] ──► 플레이
```

세 입구 모두 **같은 GameSpec**으로 수렴한다. 엔진은 입구를 모른다(CLAUDE.md §1-❹).
템플릿에 없는 완전히 새로운 유형만 별도의 **실험적 코드 생성 경로**(느림·플래그 표시)로
분기. 기본은 항상 템플릿 경로.

## 3. 단일 계약: GameSpec

`src/game-viewer/schema/gameSpec.ts` 참조. 핵심:

## 3.5 진입 방식 (3 입구 — 확정: "C")

프롬프트는 **유일한 시작점이 아니다.** 세 입구를 모두 1급으로 지원하고, 전부 동일 GameSpec으로 수렴한다.

### ① 템플릿 갤러리에서 시작 (기본 화면)
교사가 게임 유형을 먼저 고른다. 유치원 현장은 "오늘 과일 세기 활동"처럼 **유형이 먼저 정해지는**
경우가 많다. 빈 프롬프트창보다 발견성·인지부담에서 압도적으로 유리 — 갤러리가 "이런 것도 되는구나"를
보여주며 시작시킨다.

**선택 후 콘텐츠 채우기 = 폼 우선 + 프롬프트 옵션:**
- **간단 폼**으로 기본 완성: 카테고리 탭("무엇을? [과일][동물][탈것][직업]..."), 개수/난이도, 연령.
  글 입력 없이 탭만으로 끝까지 간다.
- 원하면 **그 안에서 자유 프롬프트로 미세조정**: "우리 반 텃밭 채소들로".

> 🔑 **폼 경로는 LLM 없이 동작.** `buildSpecFromForm()`이 선택값을 GameSpec으로 결정적 조립:
> 보기 숫자는 산술 생성, 아이템은 `contentSets.ts`의 카테고리별 큐레이션 셋에서 OpenMoji ref로 해석.
> → 프롬프트 경로보다 빠르고 안정적. 미세조정/자유 프롬프트만 LLM(Router→전문 에이전트)을 탄다.

### ② 프롬프트로 시작 (빠른 길)
"동물원 동물 세기 게임 만들어줘" → Router가 템플릿+콘텐츠 매핑. 상단 **퀵픽 칩**으로 빈 입력 부담 완화.

### ③ 예시에서 변형 (M3)
스타터 라이브러리의 기존 게임을 열어 소재만 교체. "이 사자 세기 게임을 우리 반 토끼로" — 교사 이미지
교체(§6)와 자연스럽게 연결.

### 구현 영향: 엔진 0, 입력 UI만 추가
- 신규: `entry/`(갤러리·폼·프롬프트 바·예시), `generate/buildSpecFromForm.ts`, `generate/templateForms.ts`,
  `generate/contentSets.ts`.
- **엔진·템플릿 렌더러는 변경 없음.** 모든 입구가 GameSpec을 만들면 끝.
- 폼 정의는 **데이터 주도**: 새 템플릿이 자기 폼 필드를 선언하면 갤러리·폼이 자동 확장.

## 4. 게임 템플릿

### 4.1 counting — 숫자 맞추기 (M1)
- **메커니즘**: 아이템 N개를 흩뿌림 → 탭할 때마다 **통통 튀며 카운트업 음성** → 숫자 보기 선택.
- **핵심**: "세는 행위 자체"에 피드백. (탭 → bounce + "하나!", "둘!"...)
- **에셋**: OpenMoji (동물·식물·탈것·음식·과일·사물 전부 가능).
- **라이브러리**: Motion(흩뿌림·bounce), Howler(카운트 음성), confetti(정답).

### 4.2 silhouette — 실루엣 맞추기 (M1)
- **메커니즘**: 검은 실루엣 표시 → 보기 중 선택 → 정답 시 **실루엣이 컬러로 모핑되며 등장**.
- **핵심**: SVG라 실루엣 변환이 공짜(알파→단색). 교사 사진도 동일 처리로 들어옴.
- **에셋**: OpenMoji SVG.
- **라이브러리**: Motion(컬러 모핑·스케일 인).

### 4.3 emotion — 감정 공감 (M2, ★ 쇼케이스)
- **메커니즘**: Rive 캐릭터가 감정을 **연기** → 아이가 감정 식별 → **공감 반응 단계**.
  (슬픈 캐릭터 → "안아주기" 탭 → 캐릭터가 위로받으며 표정이 밝아지는 애니메이션)
- **핵심**: 정서 교육 가치 최고 + 기술 차별점 최대. 정적 이미지로 불가능.
- **에셋**: Rive 상태머신 (감정별 + 반응 전이 상태).
- **라이브러리**: Rive(상태머신), Motion(보조).

### 4.4 matching — 선잇기(개념 연결) (M2)
- **메커니즘**: 좌/우 칼럼(동물–먹이, 직업–도구 등)을 **드래그로 연결**, 스냅 + 정/오답 피드백.
- **핵심**: 개념적 유사성 학습. 관계 라벨로 난이도 보정.
- **라이브러리**: Konva(드래그·히트검출·선), Motion(스냅 피드백).

### 4.5 확장 (같은 GameSpec 구조로 계속 추가)
짝맞추기(memory), 분류하기(sorting), 순서 맞추기(sequencing), 같은 그림 찾기, 미로 등.
**템플릿=엔진, 프롬프트=콘텐츠 주입기** 구조라 무한 확장.

## 5. 생성 파이프라인 & 속도

속도의 핵심 = **흔한 케이스에 코드·이미지를 생성하지 않는 것.**

1. **에셋 로컬/CDN** (OpenMoji) → 즉시, 생성 0
2. **TTS 캐시** — 같은 문장 재사용
3. **게임 캐시** — (templateId + 정규화 파라미터) → 재사용
4. **스타터 라이브러리 ~50개 사전 생성** → 많은 프롬프트가 캐시 히트 (M3)
5. **프로그레시브 렌더** — 게임 셸을 즉시 띄우고 콘텐츠 스트리밍

**목표**: 에셋 라이브러리 경로 **1초 미만**. 생성 폴백만 수 초.

## 6. 교사 이미지 교체 (M3, 차별화 핵심)

교사가 자기 반 아이 사진·교실 사진·직접 그린 그림·화분 사진을 게임 에셋으로.
"우리 반 토끼 인형이 몇 개?"가 "어떤 동물?"보다 훨씬 강력하다.

### 6.1 처리 파이프라인 (업로드 → 엔진 품질 에셋)
1. **배경 제거** — `@imgly/background-removal` (브라우저 온디바이스, 서버비 0, 프라이버시 유리).
   폴백: remove.bg / Replicate(BiRefNet).
2. **정규화** — 정사각 캔버스, 일정 비율·여백으로 시각 무게 통일.
3. **실루엣 변환** — 누끼 알파 → 단색. 사진도 실루엣 게임에 그대로.
4. **스타일 통일(선택)** — 1순위: 모든 에셋에 동일 둥근 프레임/그림자(가볍고 안전).
   고급: 사물·식물만 선택적 일러스트화. **사람·아이 사진은 누끼+프레임까지만.**

### 6.2 안전성 (최우선)
- 업로드 이미지 **반드시 안전 분류기 통과** → 실패 시 `status: "rejected"`, 진입 차단.
- 아이 사진 처리 **온디바이스 우선** (외부 전송 최소화).
- **동의·보관 정책** 제품 차원 필요. **기관 격리 저장**(Supabase 기관별 버킷), 세션 한정 우선.
- 아이 얼굴 **일러스트화/변형 기본 비활성**. 원본 보존 + 누끼 + 프레임이 안전·충분.

### 6.3 교사 UX
- **A. 인라인 슬롯 교체** — 게임 생성 후 각 에셋 위 "이미지 바꾸기" → 업로드/촬영 → 자동 처리 → 슬롯 주입.
- **B. 프롬프트 직접 지시** — "우리 반 사진들로 같은 그림 찾기" → 업로드 묶음을 AI가 에셋 배치.
- **내 에셋 보관함** — 처리한 이미지(반 마스코트 등)를 기관 라이브러리에 저장, 재사용.

### 6.4 처리 시간
배경 제거가 병목(이미지당 1~3초). 대응:
- 저해상 미리보기로 슬롯 먼저 채우고 백그라운드 처리.
- 처리 에셋은 보관함 캐시 → 재사용 시 즉시.
- 일괄 업로드는 병렬 + 진행률 표시.

## 7. 보드/툴바 통합 & 수업 모드

- **툴바 버튼** → 프롬프트 입력(상단에 템플릿 **퀵픽 칩**으로 빠른 시작) → 생성 게임이
  **React Flow 아티팩트 노드**로 보드 등장 (Workflow Lane 정합).
- 보드에서 **인라인 플레이**, 저장·재생·파라미터 수정·공유.
- **수업 모드(전체화면)**: 프로젝터·교실 디스플레이 최적화. 크게 띄우고 교사 컨트롤
  (다음/힌트/리셋). 교실 빔 출력은 킨더버스 핵심 시나리오 → 1급으로 다룸.

> **디자인 경계**: 보드 노드 프레임·툴바·프롬프트 입력(교사가 보는 부분)은 **Milray Park**
> 유지. **게임 플레이 화면 안쪽**(아이가 보는 부분)은 Milray Park를 적용하지 않고 **파스텔/큐트
> 토큰**(`src/game-viewer/theme.ts`, CLAUDE.md §5)을 쓴다. 두 시스템의 목표가 다르기 때문.

## 9. 마일스톤 & 수용 기준

### M1 — 엔진 + 2 템플릿 (OpenMoji-only)
**범위**: GameSpec + zod 검증 + GameViewer/GameShell + `counting` + `silhouette` +
Motion + Howler + CLOVA 스텁 + confetti + **3 입구 중 ①템플릿 갤러리(폼 시작, LLM 없음)와
②프롬프트(목업) 둘 다 동작**.
**수용 기준**:
- [ ] `examples.ts`의 spec을 GameViewer에 넣으면 게임이 렌더된다.
- [ ] counting: 아이템 탭 → bounce + 카운트업 음성 → 숫자 선택 → confetti + 칭찬 음성.
- [ ] silhouette: 실루엣 표시 → 정답 선택 → 컬러 모핑 등장 → 보상.
- [ ] 오답 시 부정 연출 없이 다정한 재시도.
- [ ] **템플릿 갤러리 → 유형 선택 → 간단 폼(카테고리·개수·연령) → `buildSpecFromForm()`(LLM 없음)
      → 게임 플레이**가 끝까지 동작한다.
- [ ] 폼 안에서 (옵션) 프롬프트 입력 자리가 있고, 비워도 폼만으로 완성된다.
- [ ] 프롬프트 입력(목업 매핑이라도) → 게임 생성 루프가 돈다.
- [ ] 모든 지시/라벨/피드백이 음성으로 나온다(스텁 음성이라도).

### M2 — 차별화 템플릿
**범위**: `emotion`(Rive 상태머신 + 공감 반응) + `matching`(Konva 드래그·선잇기).
**수용 기준**:
- [ ] emotion: 캐릭터 감정 연기 → 식별 → 공감 액션 → 캐릭터 표정 전이.
- [ ] matching: 좌우 드래그 연결 → 스냅 → 정/오답 피드백.
- [ ] 실제 생성 에이전트(Router→전문 에이전트)가 프롬프트로 4 템플릿 GameSpec 생성.

### M3 — 규모·완성도
**범위**: 교사 에셋 파이프라인(누끼+슬롯교체+안전검증+기관 보관함) + 캐시 라이브러리
~50개 + 수업 모드 폴리시 + 이미지 생성 폴백.
**수용 기준**:
- [ ] 교사 이미지 업로드 → 배경제거 → 슬롯 주입 → 게임 플레이.
- [ ] 안전 분류기 거부 케이스가 진입 차단된다.
- [ ] 수업 모드 전체화면 + 교사 컨트롤.
- [ ] OpenMoji 미보유 소재가 생성 폴백(스타일 락 + 안전)으로 채워진다.

## 10. 비고: 다국어

킨더버스의 일본/동남아 확장 맥락상 `ttsLocale`을 `ko-KR`/`ja-JP`/`en-US`로 설계.
라벨을 i18n key로 승격하면 동일 GameSpec이 다국어로 재생된다. (M3 이후)
