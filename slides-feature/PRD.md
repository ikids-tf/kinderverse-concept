# PRD — KinderVerse 슬라이드 생성 기능

## 1. 배경 / 목표

유치원·어린이집 교사는 수업, 부모설명회, 행정 보고에 슬라이드를 자주 만들지만 디자인·제작에 많은 시간을 쓴다. 킨더버스 안에서 **한 줄 요청으로 브랜드 품질의 슬라이드를 자동 생성**하고, 캔버스에서 쉽게 다듬어 PDF/PPTX로 내보내는 기능을 제공한다.

핵심 가치 두 가지: **쉬워야 한다**(교사가 디자인을 만질 필요 없음), **예뻐야 한다**(시스템이 브랜드 품질을 보장). 효율화 축은 속도·퀄리티·비용이며, 비용·속도의 대부분은 이미지 생성에서 갈리므로 최적화는 이미지 레이어에 집중한다.

## 2. 사용자 & 3대 용도

| 용도 | 대상 | 특징 |
|---|---|---|
| **수업용 (lesson)** | 3~5세 아동 | 큰 삽화, 글자 최소, 채도 높게, 놀이 흐름 |
| **부모설명회용 (parent)** | 학부모 | 따뜻 + 신뢰, 발달·출결 차트, 근거 제시 |
| **기타행정용 (admin)** | 내부/기관 | 깔끔·기능적, 텍스트·구조 위주, 삽화 없음 |

## 3. 핵심 설계 원칙 (불변식 — CLAUDE.md와 동일)

1. 텍스트는 절대 이미지 안에 굽지 않는다. 모든 글자는 엔진이 렌더.
2. Claude는 DeckSpec JSON만 생성. 렌더링은 엔진 전담.
3. 슬라이드 엔진은 React 컴포넌트 렌더러(HTML/CSS → PDF/PNG/PPTX). 외부 SaaS 금지.
4. 레이아웃은 enum으로 고정. 자유 디자인 금지.

## 4. 파이프라인 아키텍처

```
교사 자연어 요청 (+ 용도·연령 칩)
   │
[Router 에이전트]  용도 분류(lesson/parent/admin) + 길이·연령 추출
   │
[장표 에이전트]  Claude가 DeckSpec JSON 생성 (레이아웃·블록·이미지 프롬프트 포함)
   │   ├─ admin  → 이미지 생성 없음
   │   ├─ parent → 차트(Recharts) + 선택적 히어로(캐시 우선)
   │   └─ lesson → [이미지 워커] 캐시 조회 → 없으면 생성(style-lock) → Supabase
   │
[슬라이드 엔진]  DeckSpec + 에셋 → React 템플릿 렌더
   ├─ My Board 캔버스에 슬라이드 노드(워크플로우 레인)
   └─ Export: PDF(M1) / PNG / PPTX(M2)
```

장표 에이전트는 **워크플로우 레인과 1:1로 매핑**된다: 한 줄 요청 → 아웃라인 노드 1개 → 슬라이드 노드 N개가 가로 확장.

## 5. 데이터 모델 — DeckSpec

`deckspec.schema.json`이 정식 계약이다. 핵심 형태:

```jsonc
{
  "category": "lesson | parent | admin",
  "theme": "milray.light | milray.warm",
  "ratio": "16:9",
  "ageBand": "3세 | 4세 | 5세 | 혼합",
  "title": "나비의 한살이",
  "slides": [
    {
      "layout": "hero-image",
      "blocks": [
        { "type": "title", "text": "나비의 한살이" },
        { "type": "image", "role": "hero",
          "prompt": "노란 호랑나비, 봄 꽃밭, 부드러운 윤곽", "assetId": null }
      ],
      "speakerNote": "아이들에게 나비를 본 적 있는지 먼저 물어보세요."
    }
  ]
}
```

레이아웃을 enum으로 고정하는 것이 품질의 핵심이다. Claude는 정해진 카드 안에서만 고르므로 결과가 항상 예쁜 템플릿 안에 들어온다.

## 6. 레이아웃 카탈로그

| layout | 용도 | 주 블록 |
|---|---|---|
| `title` | 표지/섹션 구분 | title, subtitle |
| `hero-image` | 큰 삽화 1장 + 짧은 제목 | title, image(hero) |
| `big-text` | 한 문장 강조 | title(대형) |
| `two-column` | 글+그림 또는 글+글 | body, image/body |
| `bullets` | 요점 목록(최대 7) | title, bullets |
| `photo-grid` | 활동 사진 모음 | image×N |
| `quote` | 인용/메시지 | quote, caption |
| `chart` | 데이터 시각화 | chart(bar/line/pie/radar), caption |

## 7. 용도별 차별화 & 효율화

| 항목 | lesson | parent | admin |
|---|---|---|---|
| 삽화 밀도 | 슬라이드당 1 풀삽화 | 히어로 선택적 | 없음(아이콘만) |
| Claude 모델 | Sonnet | Sonnet | Haiku |
| 이미지 생성 | 풀(최고 비용) | 캐시 우선·선택적 | 없음 |
| 속도 | 느림→캐시로 보완 | 중간 | 가장 빠름 |
| 주 레이아웃 | hero-image, big-text | two-column, chart, quote | bullets, title, chart |

효율화 전략(전부 이미지 레이어에 집중):
- **스타일 락(style-lock):** 덱 단위 고정 스타일 접미 프롬프트를 모든 이미지에 강제 → 한 덱 안 화풍 통일.
- **에셋 라이브러리 + 재사용:** 생성 삽화를 Supabase에 태그(주제·계절·사물) 적재. 쓸수록 신규 생성↓ → 비용 우하향. 이미지 LOD 인프라 재사용.
- **마스코트 캐릭터 일관성:** 레퍼런스 1~3장(정면·3/4·전신)으로 반복 캐릭터 고정.
- **모델 추상화:** `ImageProvider` 인터페이스로 교체 가능. 텍스트를 안 굽기로 했으므로 한글 렌더 품질 불요, 삽화 품질·일관성·단가로만 선택.

## 8. 킨더버스 통합

- **에이전트:** 3-tier(Router/specialist/tools)에 `장표` 전문 에이전트 추가. 라우터가 슬라이드 의도 감지 시 라우팅.
- **워크플로우 레인:** 슬라이드 생성을 레인으로 표현(아웃라인→슬라이드 노드 가로 확장).
- **My Board 캔버스:** 슬라이드를 React Flow 노드로. 인라인 편집은 TipTap, 순서는 드래그, 무한 undo는 zundo.

## 9. 교사 UX

- 입력은 **한 줄 + 칩 2개**(용도·연령). 나머지(길이·톤·레이아웃)는 Claude가 결정. 빈칸 폼 금지.
- **원샷 생성 → 캔버스 편집.** 텍스트 인라인 수정, 이미지 "다시 그리기/교체", 드래그 정렬.
- 예쁨은 시스템이 보장. 모든 출력이 Milray 토큰 안에서만 렌더되어 교사가 못생기게 만들 수 없다.

## 10. Export

- **PDF (M1, 1순위):** 현장 투사·인쇄. Puppeteer/Playwright로 React 슬라이드 서버사이드 렌더.
- **PNG:** 슬라이드 단위 이미지(미리보기·썸네일).
- **PPTX (M2):** pptxgenjs로 DeckSpec → 네이티브 PPTX 매핑. PowerPoint/한컴에서 편집 가능.

## 11. 비기능 요구

- **성능:** admin 덱은 이미지 생성 없이 수초 내. lesson 덱은 캐시 적중 시 빠르게, 미적중 시 이미지 생성 시간 지배.
- **비용:** 이미지 생성 호출 수 = 비용. 캐시·재사용·카테고리 분기로 최소화. admin은 0.
- **프라이버시:** 아동·교사 데이터는 킨더버스/Supabase 안에서만 처리. 외부 유출 금지.
- **접근성:** 명도 대비, 큰 폰트 옵션(특히 lesson 투사 환경).

## 12. 마일스톤

- **M1:** 자체 엔진 + DeckSpec 파이프라인 + parent/admin 템플릿 + Recharts 차트 + **PDF export**. 한 줄 요청→완성 덱 루프 닫기.
- **M2:** 수업용 style-lock 삽화 파이프라인(`ImageProvider`·캐시·마스코트) + **PPTX export**. 이미지 모델 후보 A/B 후 확정.
- **M3:** 다국어(ja/en), 편집 UX 폴리시, 4:3 옵션, 템플릿 확장.

## 13. 성공 지표

- 교사가 디자인 손 안 대고 바로 쓸 수 있는 덱 비율.
- 한 줄 요청→첫 완성 덱까지 소요 시간(카테고리별).
- 슬라이드당 평균 이미지 생성 호출 수(캐시 적중률로 우하향).
- 생성 후 교사 편집량(적을수록 좋음).
