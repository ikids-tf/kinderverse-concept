## 0. 한 줄 요약

킨더보드가 "문서 단위"였다면 **킨더버스는 "공간 단위" 워크스페이스**다. 교사가 자연어로 의도를 말하거나 보드 위 대상을 선택해 명령하면, 얇은 3계층 에이전트가 의도를 해석해 전문 에이전트·도구로 라우팅하고, 결과는 컴포넌트 레지스트리로 제약된 AUI로 보드/페이지에 렌더된다. 데이터가 쌓일수록 똑똑해진다.

## 1. 프로토타입 분석 & 인사이트

### 1.1 실제 빌드 화면에서 확인된 자산 (계승 · 스크린샷 기준)
- **마이보드(My Board)** = 자유형 크리에이티브 보드. 좌측 세로 **생성 툴바**, 멀티컬러 스티키, 콘텐츠/AI생성 카드, 주제 핀, 줌 컨트롤, **수업 모드** 버튼.
- **My Verse** = (별도 메뉴로 존재) 작업 캔버스. → **My Board로 통합(이 PRD 기준)**.
- **프롬프트바** = 채팅아이콘·+·입력·별(즐겨찾기/실행)·모드토글 + 상단 **즐겨찾기 카드**(놀이계획/놀이기록/관찰기록/문장생성/스튜디오).
- **AI 채팅** = "Claude 기반" **전용 페이지**(새 대화·추천 질문·최근 대화·추천 칩·면책 문구).
- 디자인은 이미 **크림 캔버스 + 코랄 악센트**(Milray Park) → 다크 제외 방침과 일치.
- 캔버스 인터랙션(팬·자유배치·줌·Fit·clip-path 확대) 계승.

### 1.2 프로토타입의 한계 (재설계에서 해결)
| 한계 | 해법 |
|---|---|
| 계획 없이 만들어 구조 일관성 부족 | §4 IA, §11 데이터 모델 |
| 에이전트 라우팅·계약 미정 | §5 3계층 + 출력/선택 계약 |
| UI 정적/생성형 경계 불명확 | §6 레지스트리 제약 AUI |
| 모델 비용·선택 전략 부재 | §7 라우팅·캐스케이드 |
| "똑똑해짐"이 느낌뿐 | §8 자가고도화 4-루프 |
| 아동 사진 거버넌스 부재 | §12 거버넌스 |
| My Verse/My Board 중복 | §4 통합 |

### 1.3 핵심 인사이트
1. 공간 + 자연어 + **선택(대상)** 이 정체성. 정적 셸과 생성형 결과를 분리.
2. 교사는 "끝내야 할 일"로 생각 → 의도 라우팅 + 즐겨찾기 카드.
3. 경쟁우위는 **누적 데이터 폐루프**.

### 1.4 ★ 교사 사용성 비판적 점검
| # | 리스크 | 대응 |
|---|---|---|
| R1 | 무한 캔버스 길 잃음 | 미니맵·Fit·홈·검색 등 **가장자리 컨트롤**(§4.3) |
| R2 | 빈 입력창 막막함 | **즐겨찾기 카드**(놀이계획 등) + 추천 질문 |
| R3 | AI가 관찰 지어냄 | 무근거 생성 금지 + 근거·영역 표시(§5.5/§12) |
| R4 | 결과 매번 다르게 보임 | 레지스트리 컴포넌트 형태 고정(§6) |
| R5 | 모바일/태블릿 | 태스크 우선 리스트 폴백(§6.3) |
| R6 | 지연감 | 스트리밍 + 낙관적 UI + 고속 모델 우선 |
| R7 | 콜드스타트 | 시작 템플릿/추천 질문(§8.3) |
| R8 | 과잉 자동화 불안 | 자율성 게이트 + "검토 후 내가 확정"(§9) |
| R9 | **컨트롤 과다 = 새 인지부하** | 항상 보이는 컨트롤 4종 최소화, 나머지는 **상황 시 등장**(§4.3) |

## 2. 제품 비전 & 목표

**비전**: "교사가 말하거나 고르기만 하면, 우리 원의 맥락을 아는 동료가 만들어 준다 — 확정은 언제나 교사가."

**1차 목표(6개월)**: 4대 태스크(놀이기록/관찰기록·놀이계획·통신문/문장·스튜디오) 자연어 생성+편집 / 라우팅 정확도 ≥90% / 수정 없이 채택률 30%→55%+.

**비목표(1차 제외)**: 킨더뷰 심층 통합 / 영상 풀기능(프리뷰 게이팅) / 모델 파인튜닝(RAG·메모리 대체).

## 3. 타깃 사용자 & 핵심 시나리오

**Primary persona** — **만 0~1·2·3·4·5세** 담임 교사. 영아반(0~2, 표준보육과정: 일상·기본생활습관 중심) / 유아반(3~5, 누리과정: 놀이중심). 종일 사용, 행정 부담, 모바일/태블릿 빈번.

**Job Stories**
1. 하원 후 사진 던지고 → "오늘 햇살반 블록놀이 관찰기록" → 연령 맞춤 초안 + 사진 자동 배치.
2. "다음 주 봄 주제 놀이계획" → 주안 카드.
3. "현장학습 가정통신문, 우리 원 톤" → 통신문 초안.
4. "봄꽃 색칠 도안 A4" → 스튜디오 이미지/도안.
5. (영아) "지호 이번 주 수유·낮잠 패턴 정리" → 영아 일상 기록.
6. 보드에서 카드 2~3개 선택 → "이거 묶어서 기록 한 장으로" → 병합 생성.

## 4. 정보구조(IA) & 화면 구조 (실제 빌드 반영)

### 4.1 LNB (좌측 아이콘 레일 — 실제 화면 기준)
```
├─ 홈            ← 랜딩/대시보드
├─ 갤러리        ← 사진·이미지·도안 자산 (분류·기억 엔진 결과)
├─ My Board      ← ★ 통합 캔버스 (구 My Verse 흡수)
├─ 우리반        ← 아동 등록·관리 (4.4)
├─ 캘린더        ← 일정 (계승)
├─ 폴더          ← 완성/진행 산출물 저장·정리
└─ 프로필(하단)  ← 계정·설정
        + AI 채팅 (상단/전역 진입 — 전용 페이지, 4.5)
```
> **My Verse 메뉴 완전 제거.** 별도 메뉴 없이 My Board로 단일화하며, 마이버스의 작업 캔버스 기능은 My Board 툴바의 '프레임/캔버스 추가'가 흡수(4.3). 기존 My Verse 데이터/보드는 마이그레이션으로 My Board에 편입.

### 4.2 My Board = 통합 캔버스 (핵심)
자유형 보드 + 작업 캔버스가 한 공간에 공존. "모으고·배치하고·완성한다."
- **자유 프리미티브**(툴바로 추가): 스티키 메모(멀티컬러), 텍스트, 이미지카드, 문서/메모 카드, 영상, 도형.
- **작업 캔버스/프레임**(구 My Verse): 놀이기록·놀이계획 등 *작업 자체*를 수행하는 캔버스를 보드에 얹음.
- **AI 생성 카드**: "선택 후 프롬프트로 AI 생성" — 선택한 대상/프롬프트로 결과 생성(§5.5).
- **워크플로 레인(★ 핵심)**: 한 번 요청하면 연관 자료·문서가 **가로로 한 레인 안에 순서대로 자라남**(§4.2.1).
- **주제 핀/존**: 보드 제목("우리 반 이야기")·주제 핀("이번 주 주제·가을")으로 구역화.

### 4.2.1 ★ Workflow Lane — 보드의 중심 상호작용
교사가 보드에서 **하나를 요청하면, 그와 연관된 자료/문서가 한 섹션(레인) 안에서 왼→오로 순서대로 펼쳐진다.** 채팅 로그가 아니라 *공간에서 작업 흐름이 자라는* 방식 — 킨더버스의 핵심 차별점.
- 예: "5월 가정의 달 활동 추천" → **놀이계획 레인** 생성 → ① 아이디어 리스트 →(복수선택)→ ② 활동 사진(AI 개념 이미지) → ③ 계획안(편집) → ④ 활동지(편집·저장) → **[레인 저장]→ 폴더에 번들**(계획안+활동지+이미지+연결자료).
- **템플릿 기본 + 라우터 동적 제안**: 단계 순서는 템플릿으로 고정(예측가능), 각 단계 후 라우터가 맥락 기반 `suggested_next`(예: "가정연계 통신문")를 *선택지*로 제안. **제안 레이어는 초기부터 탑재.**
- **진행은 클릭으로만**(자동 전체 실행 금지), **선택이 다음 단계 입력**, 모든 노드는 인라인+프롬프트바 편집.
- 구현은 **새 에이전트 없이 '워크플로 러너'** 한 겹이 기존 전문 에이전트(계획/스튜디오/문장/기록)를 템플릿 순서로 호출. 상세 명세·데이터 모델은 SKILL.md §9.
- 안전: ②의 활동 사진은 **AI 생성 개념 이미지**(실제 아동 사진과 분리, "AI 생성" 라벨). 발송·삭제는 자율성 게이트.

### 4.3 My Board 컨트롤 (스크린샷 기준 + R1/R9 보강)
**좌측 세로 툴바** (실제 화면):
| 도구 | 기능 |
|---|---|
| 선택(커서) | 대상 선택·복수 선택(=컨텍스트, §5.5) |
| 문서/페이지 | 문서·페이지 카드 추가 |
| 이미지 | 이미지카드 추가 |
| 텍스트(T) | 텍스트 블록 추가 |
| 메모/카드 | 스티키 메모·카드 추가 |
| 영상 | 영상 카드 추가 |
| 도형 | 도형/스티커 추가 |
| **프레임/캔버스** | **작업 캔버스(구 My Verse) 추가** ← 통합 핵심 |
| (하단) 잠금/홈 | 보드 잠금·홈 위치 복귀 |

**탐색/정리 컨트롤** (R1·R9 — 항상 보이는 건 최소화):
- 항상: **미니맵·Fit(맞춤)·검색·+추가**.
- 줌 컨트롤(우하단, —/82%/+/맞춤) · **수업 모드**(우상단, 보드를 교실 투사용 발표 뷰로 — 슬라이드쇼·수업 기록 포함 상세 §16.8).
- 상황 시 등장: **선택 툴바**(카드 선택 시 묶기·정렬·삭제·생성), 정렬/그리드 스냅, 존 만들기.

### 4.4 '우리반' 모듈
아동 등록·관리. 아동별 사진/문서 모아보기 + **특이사항·투약·출결석·하원시간**·메모. 영아/유아·반 편성. → §8 메모리·§5 슬롯의 1차 컨텍스트 소스.

### 4.5 'AI 채팅' 전용 페이지 (스크린샷 기준)
- 헤더: "AI 채팅 · Claude 기반".
- 좌측 패널: **+ 새 대화**, **추천 질문**(예: 가정통신문 초안·실내놀이 추천·관찰일지 누리과정 정리·안전교육 자료), **최근 대화**.
- 메인: "무엇을 도와드릴까요?" + 추천 칩 + 동일 프롬프트바.
- 푸터: **면책 문구**("AI는 실수할 수 있습니다 — 확인 필요").
- 역할: 자유 대화 + 새 작업 시작(어느 페이지든 열기). 작업 페이지 안에서는 프롬프트바가 그 페이지를 핸들링(§6.2).

### 4.6 갤러리 / 폴더 / 캘린더
갤러리=자산 라이브러리(분류 결과) · 폴더=산출물 저장·정리 · 캘린더=일정→생성 트리거 연계.

**상태 계층**: 전역(테넌트/교사) · 보드(위치/줌/배치/선택) · 태스크(진행 중 생성/편집). Zustand.

## 5. 에이전트 아키텍처 (핵심)

### 5.1 "얇은 3계층" + 공유 Pedagogy Foundation
```
        ┌───────────────────────────────────────────┐
        │ Pedagogy Foundation (공유 토대)             │
        │ 0~2 표준보육과정 · 3~5 누리과정 · 발달적합성 │
        │ 영역 연계 매핑 · 무근거 생성 금지            │
        └───────────────────────────────────────────┘
                ▲ 모든 Tier1 상속
[입력/선택]
   │
Tier 0 ── 라우터 — 빠른·저가 모델
   │  의도분류 / 슬롯추출 / **선택 컨텍스트** / 라우팅 / 확신도 (생성 X)
   ▼
Tier 1 ── 전문 에이전트 (기능명 통일, 적합성은 공유 레이어가 보장)
   ├─ 기록     : 관찰기록(영역분석·평가용) + 놀이기록=놀이이야기(사진배치·활동서술·학부모발송) (+ KB 오너)
   ├─ 계획     : 놀이계획 / 주안·월안 (활동 맥락을 스튜디오에 공급)
   ├─ 스튜디오 : 이미지·영상·도안 + 활동지/워크시트
   └─ 문장     : 문장생성 · 가정통신문·공지·평가서
   ▼
Tier 2 ── 결정형 도구
   ├─ 이미지/영상 생성 · 문서 템플릿 엔진
   └─ 분류·기억(엔진) : 사진 **실시간 분류**(기개발 API) + 임베딩·RAG
```
**즐겨찾기 카드 → 에이전트 매핑**: 놀이계획=계획 · 놀이기록/관찰기록=기록 · 문장생성=문장 · 스튜디오=스튜디오.
> 라우터는 시스템 내부 컴포넌트라 기능명("라우터")으로 통일. (내부 코드 식별자 예: `router`, `agent.record`, `agent.plan`, `agent.studio`, `agent.writing`, `engine.curator`.)

**산출물 분기·협업 (상세는 SKILL.md §1·§4.1)**
- **기록 2모드**: `observation`(관찰기록, 발달·영역분석, 행정/평가용) / `story`(놀이기록=놀이이야기·활동기록, 사진 배치+활동 서술, **학부모 발송용**). 놀이기록은 분류·기억 엔진이 사진 선별 → 기록이 서술, 부모 톤은 문장 에이전트 차용. 발송=L2/L3.
- **활동지**: 만드는 주체는 항상 스튜디오. (A) **놀이계획 연결** — 계획이 활동 맥락(연령·영역·목표)을 공급, 결과를 계획에 link. (B) **독립** — 라우터가 곧장 스튜디오로. 출력은 `WorksheetCard`(A4·인쇄·다운로드).

> **(구현 반영)** Tier1에 **'디자인'(디자인 디렉터)** 에이전트가 추가되었다 — 콘텐츠를 생성하지 않고 프레임 배치 variant·장식 스티커·표지 대상만 결정하는 레이아웃 전담(저가 모델, 실패 시 규칙 기반 폴백). 기능명 원칙 준수. 상세 §16.0·§16.4·§16.11.

### 5.2 유아교육 적합성은 '공유 레이어'가 보장
계획·스튜디오·문장 포함 **모든 전문 에이전트가 Pedagogy Foundation을 상속**한다. '기록' 에이전트가 다른 에이전트를 직렬 검수하지 않는다(지연·병목·단일실패점 방지). '기록'은 지식베이스 *오너*. **고위험 산출물(평가서)에만** 출력 하네스의 **자동 적합성 검증 패스**(체크리스트) 1회.

### 5.3 에이전트 명명
**기능명으로 통일** — 라우터 / 기록 / 계획 / 스튜디오 / 문장 / 분류·기억. 교사 노출 라벨은 즐겨찾기 카드 그대로(놀이계획·놀이기록·관찰기록·문장생성·스튜디오), 내부 식별자는 위 기능명. (이전되는 핵심 가치는 라우터 + 2단계 패턴이며, 페르소나 이름은 사용하지 않는다.)

### 5.4 왜 3계층인가
단일콜=정확도·디버깅 악화 / 풀 멀티에이전트=지연·비용 폭증 / 3계층=라우팅·생성·도구 책임 분리.

### 5.5 ★ 출력 계약 + 선택=컨텍스트 + 안티-환각
프롬프트바는 **현재 페이지 + 선택 대상**을 항상 라우터에 동봉한다. "선택이 곧 범위."
```jsonc
{
  "page": "myboard",
  "selection": { "ids":["card_12","card_15"], "types":["image_card","memo"], "count":2 },
  "available_actions": ["generate","merge","relayout","restyle","add_media"],
  "intent": "merge",
  "scope": "selection",            // selection | page | new
  "route_to": "record",
  "lane": { "id":"lane_22","template":"play_plan","step":"plan","node_id":"node_3" },
  "suggested_next": [ { "action":"family_letter","label":"가정연계 통신문","reason":"5월 가정의 달","confidence":0.62 } ],
  "confidence": 0.93,
  "grounding": { "photos":["..."], "teacher_notes":["..."] }  // 없으면 생성 금지
}
```
**규칙**: ① 선택 있으면 그 대상에만 작용. ② `available_actions` 밖은 라우팅 안 함. ③ 레이아웃 변경=L1 자동(되돌리기), 생성·병합=미리보기 후 적용. ④ `confidence<0.7` 되묻기. ⑤ `grounding` 없으면 관찰 생성 금지·근거/영역 표시. ⑥ 다음액션 칩 = 기본 다음단계(템플릿, 항상) + `suggested_next`(라우터 제안 0~2, 옅게 "추천", 확신도<0.6 숨김). 제안은 선택지일 뿐 자동 실행 금지.

### 5.6 MeetFlow 로스터 대비
MeetFlow 패턴 대비: 승계=라우터·2단계. 신규=분류·기억 엔진, Pedagogy Foundation. 페르소나 이름(프뢰벨 등)은 폐기하고 기능명으로 통일.

## 6. AUI & 상호작용 전략

### 6.1 경계
| 영역 | 방식 |
|---|---|
| 셸·LNB·My Board 캔버스·툴바·갤러리·우리반·캘린더·폴더·AI채팅 셸 | **정적 UI** |
| 결과 표면(기록·계획·통신문·이미지 갤러리·AI생성 카드) | **AUI** |

### 6.2 ★ 프롬프트바 = 컨텍스트 인식 핸들러
하나의 바, **페이지마다 다른 능력**. 각 페이지가 받을 수 있는 `available_actions`를 등록 → 라우터는 그 범위에서만 라우팅.
- **AI 채팅 페이지**: 일반 대화 + 새 작업 시작.
- **작업 페이지(놀이기록 등)**: 그 작업물 핸들러(사진 추가·문단 수정·영역 재연계·톤 변경).
- **My Board**: 보드 핸들러(카드 추가·선택 대상 정렬/묶기/생성).
- **즐겨찾기 카드**(놀이계획/놀이기록/관찰기록/문장생성/스튜디오): 입력이 빈 상태에서 **별 버튼**을 누르면 바 위로 솟아오르고, 카드 클릭 시 해당 페이지로 이동. **추천 칩은 선택 상태에 따라 동적 변경**(카드 2개 선택 시 "묶기/한 장으로" 등). 좌측 **메시지 아이콘=AI 채팅 이동**, 우측 **둥근 토글=바 접기/펼치기**. 상세 동작·구현 기준은 §10.4.

### 6.3 AUI = 레지스트리 선택 + 파라미터 (임의 생성 금지)
`에이전트 JSON → 스키마 검증 → UI Registry 매칭 → 정적 컴포넌트 렌더`
**Registry 카탈로그(초기)**: `RecordDraftCard`(관찰기록)·`PlayStoryCard`(놀이기록=놀이이야기, 사진슬롯+활동서술+학부모발송)·`WeeklyPlanGrid`·`LetterPreview`·`StudioGallery`(이미지/영상/도안)·`WorksheetCard`(활동지/워크시트, A4·인쇄·다운로드·계획연결)·`AssessmentReport`(발달평가서 — 적합성 검증 결과 표시, 구현 추가)·`PhotoClusterBoard`·`StickyNote`·`TextBlock`·`ImageCard`·`WorkCanvasFrame`(구 My Verse)·`ClarifyPrompt`·`FavoriteCardRail`·`SelectionToolbar`. (확장 상세 §16.15)

### 6.4 반응형 (모든 뷰포트 + 모바일 폴백)
셸=container query(LNB 레일↔하단탭) · 결과=fluid grid · **My Board=무한 캔버스+줌/팬, 작은 화면은 태스크 우선 리스트로 폴백(R5)** · BP: sm480/md768/lg1024/xl1440/2xl1920+.

플로팅 형태. 모든 페이지에 상주하며 컨텍스트 핸들러로 동작(§6.2). 좌→우 구성: **메시지 아이콘 · + 추가 · 입력창 · 별/전송 버튼 · 둥근 토글**.

| 요소 | 동작 |
|---|---|
| **좌측 메시지 아이콘** | 어느 페이지에서든 클릭 시 **AI 채팅 페이지로 즉시 이동**(전역 바로가기). 또한 바가 닫힌 상태에서는 이 버튼만 남아 **바를 다시 펼치는 트리거**. |
| **+ 추가** | 첨부/대상 추가. |
| **입력창** | "무엇이든 물어보세요". 현재 페이지 + 선택 대상을 컨텍스트로 동봉(§5.5). |
| **별 ↔ 전송(토글)** | **입력이 비어 있으면 = 별(즐겨찾기) 버튼**, **입력이 있으면 = 전송(게시) 버튼**으로 전환. |
| **별 클릭 (입력 없음)** | 바 위로 **즐겨찾기 카드 솟아오름**(놀이계획·놀이기록·관찰기록·문장생성·스튜디오). 카드 클릭 → **해당 페이지로 이동**. |
| **우측 끝 둥근 토글** | 클릭 시 바가 **접혀 메시지 아이콘 버튼 하나만 남음**. 다시 그 버튼으로 펼침. |

스트리밍 중 인라인 진행 표시. 추천(즐겨찾기) 카드는 선택 상태에 따라 동적 변경(§6.2).

> **(구현 확장)** 생성 중 게시 버튼 호버 시 **■ '생성 중단'** 전환(§16.4-5), 입력 연동 **보관함 자산 스트립**(다중 선택→배치, §16.6), 보드 선택 칩("N개 선택") 등이 추가되었다.

> **(구현 확장 2026-06)** ① 입력 텍스트가 차면 스크롤 대신 **바가 위로 자동 확장**(긴 프롬프트도 한눈에). ② **+ 버튼 = 이미지/텍스트 문서 업로드** 후 그와 연관된 생성 요청(이미지=보드에 배치, 텍스트=메모 카드로 변환 후 프롬프트 적용). ③ **바 또는 좌측 보드 툴바 호버 시 바 불투명도 상승**(명시성 확보, 완전 불투명은 아님 — 평소 살짝 비치는 글래스). ④ **자료 선택 시 코랄 테두리 + 하단 간접광**(블루-에메랄드, 밝기 호흡 애니메이션, 코랄 라인에 정렬·가로 중앙 70%·좌우 자연 페이드). ※ 간접광 색은 코랄+골드 단일 악센트 원칙(§10.1)의 **사용자 요청 예외**로, 되돌리기 쉽게 `--kv-glow-blue/--kv-glow-emerald` CSS 변수로 격리했다.

> ⚙️ **개발 지침**: 위 프롬프트바 동작(메시지 아이콘→AI채팅 이동 / 둥근 토글→바 접기·펼치기 / 별↔전송 토글 / 별→즐겨찾기 카드→페이지 이동)은 **새로 구현하지 말고 `localhost:5173` 프로젝트의 기존 프롬프트바 컴포넌트 코드를 기준 구현체로 가져와 그대로 적용**한다. 정식 서비스에서는 이 컴포넌트를 공용 셸 컴포넌트로 승격해 전 페이지에서 재사용한다.

### 16.1 보드 내비게이션 (트랙패드 우선 입력 체계)
- **휠/두 손가락 스크롤 = 팬**(내추럴 스크롤). 단, 스크롤이 필요한 문서/프레임 위에서는 카드 내용 스크롤 우선.
- **핀치(Ctrl/⌘+휠) = 커서 중심 줌** — 지수 감속 매핑, Safari `gesturestart/gesturechange` 별도 지원.
- **미들 마우스 버튼 = 팬 전용**(카드 드래그로 오인 금지) · **Space+드래그 = 팬**.
- **더블클릭 내비**: 빈 배경=전체 맞춤(Fit) / 프레임 내부 빈 곳=그 프레임 포커스 / 카드=포커스 줌(프롬프트바 기준 중앙 정렬). **슬라이드·영상·3D·유튜브 뷰어 카드도 더블클릭=화면 중앙 포커스**(iframe은 포인터 경계를 못 넘으므로 `kv-embed-dblclick` postMessage로 부모에 전달, §16.10).
- **뷰포트 컬링 + 이미지 LOD**: 화면 밖 노드 렌더 제외(50% 마진), 줌 0.3 미만에서 이미지 카드는 플레이스홀더+제목으로 강등. 표시용 썸네일 최대 400px, 동시 디코드 4개 큐 제한(원본 dataURI는 보존).
- **카메라 애니메이션**: ease-out cubic 200~420ms(거리 비례), 토큰 기반 취소(`cancelPanAnimation`) — 생성 카메라와의 경합 방지.

### 16.2 보드 편집 — 선택·변형·연결·클립보드·실행취소
- **박스 선택**: 배경 드래그. **프레임은 박스 선택에서 제외**(내용만 끌었는데 프레임이 딸려오는 사고 방지). 수업·슬라이드 중에는 보이는 요소만 대상.
- **리사이즈 정책**: 프레임·텍스트·메모 = **프리폼(비비례) 리사이즈** — 코너 핸들, 반대 코너 앵커 고정, 텍스트 리플로. 최소 크기 일반 120×48 / 프레임 200×140. **문서 카드(`data.doc`)는 프리폼 잠금, 정비례 스케일만 허용**. 그 외 타입=중심 기준 정비례 스케일(0.3~6×).
- **회전 핸들**: 상단 원형 핸들, Shift=15° 스냅.
- **인라인 편집**: 메모 더블클릭 = **보이던 모습 그대로 제자리 편집**(뷰포트 점프·내용 잘림 없음). 텍스트·이미지 캡션·프레임 제목도 인라인. 문서 더블클릭은 포커스만(원문 마크다운 노출 금지).
- **요소 연결(링크)**: text·sticky·image·frame(임베드 카드 포함)의 좌우 포트에서 드래그→드롭. 곡선(3차 베지어) + **순번 라벨 체계**(머리=1, 줄기 2,3… / 가지 1-1,1-2…) — 슬라이드쇼 진행 순서의 근거. 연결선 클릭=해제, 연결된 포트 28px 이상 드래그=분리. **iframe 임베드 카드는 포인터 이벤트를 삼키므로 `kv:ports-hover` 커스텀 이벤트로 포트를 노출**(전용 워크어라운드).
- **빈 메모/이미지에 연결 시**: "내용/이미지를 생성할까요?" 제안 팝오버 — 자동 실행 금지(L1 원칙 유지).
- **클립보드**: 복사/잘라내기/붙여넣기(보드 내부) — 프레임 자식 포함 깊은 복사 + frameId/그룹/참조 재매핑. **붙여넣기는 커서 위치 우선**, 커서 없으면 원본 옆 계단식(+24px), 화면 밖이면 뷰 중앙.
- **Undo/Redo**: 커맨드 스택(최대 100, L1 변경만 기록). 이동·삭제·복제·프레임 감싸기·연결·색/크기/회전 전부 기록, AI 스폰 배치도 1 커맨드로 묶여 한 번에 되돌리기.
- **단축키**: Del/⌫ 삭제 · ⌘A 전체 선택 · ⌘D 복제 · ⌘G 그룹(+Shift 해제) · ⌘L 잠금 · ⌘Z / ⌘⇧Z(⌘Y) · ⌘C/X/V · ⌘+ / ⌘− / ⌘0 줌 · Shift+1 Fit · Esc 선택 해제. 편집 필드 포커스 중에는 무시.
- **자연어 화면 조작**(선택+명령, 생성 동사 없을 때 — 모델 호출 없이 즉시): 이동(240px 스텝)/크게(×1.25)/작게(×0.8)/크기 맞춤(첫 선택 앵커)/색 변경/한 줄 정렬/격자 정리.
- **빈 요소 파서**: "메모 3개", "이미지 두 장" 등 주제 없는 요청은 AI 호출 없이 빈 프리미티브 N개 즉시 추가(한글 수사 인식, 기본 1개).

### 16.3 프레임 시스템
- **멤버십**: `data.frameId`(권위) + 기하 겹침 판정(폴백). 드래그 종료 시 재판정(rebind).
- **자동 핏**: 자식의 실측 렌더 높이(`renderH`) 기준 + 패딩 28px — 문서가 길어져도 겹침 없이 프레임 확장.
- **프레임 크롬**: 제목 탭과 '정렬'·'폴더에 저장' 버튼이 **테두리 선 위에 절반 걸치는** 전문 레이아웃(좌=제목 탭, 우=[정렬][저장] 한 묶음). 제목 탭 더블클릭=이름 변경, 생성 진행 중엔 탭에 미니 스피너. 정렬·저장 버튼은 **최상위 프레임에만** 노출, 좁은 프레임에서는 라벨을 숨겨 아이콘만(제목 탭 폭 동적 계산으로 겹침 방지).
- **★ 프레임 정렬(`align.ts`)**: 프레임을 '편집 디자인된 한 페이지'로 보고 **제자리 정돈**(통째 재배치 아님). 동작: ① **소속 입양** — 화면상 서브 프레임 안에 있는 요소(중심점 기준)는 태그가 어긋나 있어도 그 프레임 소속으로 교정(서브 프레임 안 요소는 정렬 후에도 안에 유지) → ② 서브 프레임 먼저 재귀 정돈(후위 순회) → ③ **행·열 정돈**: 윗변이 가까운 카드는 같은 행으로 묶고(키 큰 카드—동영상 뷰어 등—옆/위에 둔 카드도 그 행에 합류), 행 안은 균일 거터(28px)로 좌측 패킹·윗변 정렬, 겹친 카드는 행에 끼어들고 넘치면 아랫줄로, 직전 행 여백에 들어가는 행은 끌어올려 채움 → ④ **콘텐츠 핏**: 프레임 박스를 내용에 딱 맞게(빈 공간 제거, 서브 프레임 포함). **왼쪽 상단 제목 텍스트(role 'header')는 페이지 머리글로 인식해 제자리 고정**, 본문은 그 줄 아래부터. 카드 크기는 변경하지 않음(내용 맞춤 크기 보존). 마인드맵·모션·러너·잠긴 카드는 제외. **Ctrl+Z 한 번으로 통째 복구**.
- **선택 감싸기**: 현재 선택을 프레임으로 래핑(월드박스+36px 패딩, 기존 멤버십 백업·복원).
- **★ 동반 이동(`frameMoveSet`)**: 프레임을 옮길 때 함께 가는 집합 — ① 소속 하위 전체(중첩 프레임·손주 재귀) ② 다른 프레임 소속 요소는 그 프레임이 새 프레임에 **완전 포함**된 중첩일 때만 ③ 무소속 단독 요소는 살짝만 겹쳐도 ④ 프레임은 **완전 포함**일 때만. **모션 라인(이동 애니메이션)은 항상 제외** — 연결 카드를 따라 엔드포인트가 자동으로 따라가므로 통째 이동 시 이중 이동·자유 끝점 끌림을 막는다(태깅돼 있어도 제외).
- **복사/붙여넣기 깊은 수집**: 프레임 복사 시 서브 프레임 안의 자료(손주)까지 **재귀로** 수집(이전엔 한 단계만 펼쳐 빈 서브 프레임이 붙던 버그 수정). 잘라내기도 동일.

### 16.4 생성 UX 정책 (★ 신규 핵심 정책)
1. **단일 문서 우선**: 놀이계획 요청은 기본적으로 **계획안 1장만 정교하게** 생성한다. `패키지·세트·한 번에·한꺼번에·전부·모두 (다)` 등의 키워드 또는 복수 산출물 명시(아이디어/이미지/도안/활동지/통신문 동반)일 때만 패키지(complex)로 구성. 확장은 생성 후 칩으로.
2. **A4 실사이즈 스트리밍**: 계획안은 보드 화면 상단 중앙에 **A4 세로 시트(480×679)** 가 줌 1 실사이즈로 즉시 자리 잡고, 초안이 문서 위에서부터 SSE 스트리밍된다(80ms 스로틀). **카메라가 글이 써지는 하단을 따라 하향 팬**(프롬프트바 위 150px 가드, 하향 전용 — 교사가 생성 과정을 실사이즈로 읽을 수 있게), 완료 후 문서 상단으로 복귀. 스트림 종료 후 정식 문서로 정리(runPlan) + **16:9 와이드 배너 표지** 비동기 생성(히어로 최대 높이 110px).
3. **디자인 디렉터 선배치**: 칩 확장 등 추가 생성은 **플레이스홀더 슬롯을 먼저 정렬 배치한 뒤** 그 자리에서 채운다 — "겹쳤다가 나중에 정렬"되는 시각 혼선 금지. 배치 variant: `default`(컬럼 그리드) / `gallery-first`(이미지 선두) / `hero-doc`(문서 선두). 장식 스티커는 통신문·기록·활동지·소식지에만, **계획안은 공식 문서로서 스티커 제외**.
4. **확장 칩(`nextSteps`)**: 프레임 하단 추천 칩(예: 놀이계획 → 아이디어 카드/활동 이미지/활동지/가정통신문). **클릭으로만 진행**(자동 전체 실행 금지 — §4.2.1 원칙 유지). 정정 칩으로 같은 프롬프트를 다른 의도로 재라우팅 가능.
5. **생성 중단**: 모든 생성은 **공유 AbortController** 1개를 따른다. 생성 중 프롬프트바 게시 버튼=스피너, **호버 시 ■ '생성 중단'으로 전환, 클릭 즉시 중단**(스트리밍 fetch 끊기 + 카메라 팬 취소 + 상태 초기화). 빈 초안 상태로 중단하면 시트·프레임까지 정리 제거.
6. **병렬 생성**: 동시 생성 카운터(`genActive`)로 추적, 병렬 작업은 세로 640px씩 비켜 배치. 새로고침 시 잔류 `loading` 플래그 위생화(무한 스피너 방지).
7. **선택-의도 불일치 다이얼로그**: 선택 타입과 요청 산출물이 어긋나면(예: 이미지 선택+활동지 요청) 3택 제시 — ① 그 자리에 생성(원본 옆 추가) ② 성격 바꿔 교체(제자리 변환) ③ 새 프레임 구성.

### 16.5 자료 배치 정책 (placement — ★ 신규 정책)
프롬프트·보관함·확장 칩에서 자료를 보드에 놓을 때의 우선순위:
1. **선택된 프레임 안** — 겹침 회피 그리드(24px 갭, 문서는 실측 높이로 장애물 판정) + 프레임 자동 확장.
2. 선택 없음 → **마지막 작업 프레임(최우측 최상위 프레임)의 오른쪽**에 상단 정렬, 우/하향 스캔 — 직전 작업 흐름이 끊기지 않게.
3. 프레임 없음 → **뷰포트 중앙** 나선 배치.
- 배치 결과가 화면 밖이면 **최소 이동량의 스무스 팬**으로 보이게 한다(가장자리 40px·프롬프트바 120px 여유) — 급작스러운 화면 점프 금지.

### 16.6 보관함 — 생성 이미지 자산 캐시 (★ 신규 기능)
- 성공한 생성 이미지는 **캡션 태그와 함께 IndexedDB(`image-assets:v1`)에 자동 저장**, 보드 삭제와 독립 보존. 태그당 최근 3장 유지.
- **같은 캡션 요청은 재생성 대신 보관함에서 즉시 재사용** — §7.2 비용 절감의 5번째 장치.
- **프롬프트바 보관함 스트립**: 보드에서 2자 이상 입력 시 검색(160ms 디바운스, "와/과/랑" 연결사 분해 다중 키워드 토큰화). 화면 가로 전폭, 최대 3줄(286px) 초과 시 스크롤, 소형 코너 라운드. **다중 선택** 후 그리드 중앙 '배치' 버튼으로 일괄 배치, 또는 선택 상태로 프롬프트 제출 시 **배치 후 그 카드들에 명령 적용**.

### 16.7 폴더 저장 — 프레임→폴더 번들 (★ §4.6 '폴더' 구체화)
- 프레임 '폴더에 저장' → **프레임 이름의 폴더**가 폴더 페이지에 생성. 유형별 변환: 이미지→`.jpg` · 문서→`.pdf` · 메모/텍스트→`.txt` · **중첩 프레임→중첩 폴더**(트리 보존).
- 각 폴더의 **첫 항목은 `board.board`** — 저장 당시 보드 배치 원형 스냅샷(레이아웃 JSON+표지) 썸네일.
- 파일명 규칙: 마크다운 기호(`#`, `**`) 제거 · 금지 문자(`\/:*?"<>|`) 공백화 · 40자 제한.
- 저장 피드백: 토스트 "폴더에 저장하는 중… → 저장했어요" (progress/success/error 3종, `kv:toast` 이벤트 버스, 하단 중앙 필).
- **폴더 페이지 뷰어**: 드릴인(브레드크럼) · 포맷별 실미리보기(이미지/축소 렌더 보드/보드와 동일한 편집디자인의 문서(kv-doc-md+표지)/메모 스니펫) · **풀스크린 뷰어 + ←/→ 연속 탐색 + 카운터 + Esc** · 다운로드는 뷰어 내 버튼(이미지=canvas JPEG 재인코딩, 문서=인쇄용 HTML→브라우저 "PDF로 저장", 메모=txt Blob) · `board.board` 뷰어의 **'마이보드에서 보기'**=해당 프레임으로 점프(노드 소실 시 제목 매칭 폴백+오류 토스트).
- 영속화: IndexedDB `kv:folder:v1`(600ms 디바운스) — 새로고침에도 유지.

### 16.8 수업 모드 · 슬라이드쇼 · 수업 기록 (★ §4.3 1줄 → 모듈로 확장)
- **수업 모드 알약**(우상단): **호버 시 '이전 수업'이 왼쪽으로 확장되고 '수업 모드'는 오른쪽 제자리 고정**(우측 앵커 — 화면 밖 밀림 방지). 진입 시 현재 보이는 카드 구성을 **수업 기록으로 자동 저장**(10분 내 동일 구성 재진입은 중복 기록 안 함). 활성 중 수동 저장 버튼(토스트 피드백, 1분 내 동일 구성은 시각만 갱신).
- **슬라이드쇼**: 수업 모드에서 ▶ — **연결 순번 라벨 순서**(1 → 1-1 → 2…)로 한 장씩 전체화면. ←/→ 이동·하단 내비(n/N)·Esc 종료, 모션 묶음 슬라이드는 모션 컨트롤 바로 대체.
- **이전 수업 패널**: 우측 슬라이드 인. 최신순 기록(일시·요일·자료 수·미리보기 썸네일 4개), **다시 열기**(살아있는 노드만 선택해 수업 모드 재진입), 호버 삭제, Esc/외부 클릭 닫기.
- **수업 기록 저장 정책**: 이미지 원본은 저장하지 않음(노드 id+캡션만 — 개인정보 최소화). 로컬 `kv:lessons:v1`(최근 50건) 우선 + 서버 미러 `GET/POST/DELETE /api/lessons`(프로토타입 파일DB `.kv-data/lessons.json`, 200건 FIFO — 이관 시 Supabase `lessons` 테이블로 치환). 서버 저장 실패는 조용히 무시(로컬 우선).

### 16.9 모션 패스 — 이동 애니메이션 도구 (★ 완전 신규 도구)
수업 발표·놀이 연출용 카드 이동 애니메이션.
- 3차 베지어 경로 노드: **출발/도착 원**(각 원 안에 '출발'·'도착' 라벨, 연결 시 코랄·미연결 시 흰 배경, 흰 후광으로 가독)을 카드에 연결(중심 스냅, 도착 카드는 추적 이동) + 곡선 ⅓·⅔ **조절점 둘**(독립 드래그로 S자·파도). 출발/도착·조절점 모두 **드래그=이동/연결·해제(5px 데드존), 클릭=지점 효과 패널 토글**.
- **컨트롤은 출발 카드가 연결된 때만 노출**: ▶/⏸ · ↺ 처음으로 · **배속 0.05~4×**(평소 "속도 N.NNx" 칩으로 접힘, **호버 시 슬라이더로 확장**) · 왕복 토글 · 플립 모드 순환(flip 좌우반전 / rotate 경사 회전 / none).
- **이미지 방향 자동 분석**: 비전 Q&A로 피사체가 보는 방향을 판정 → 진행 방향에 맞춰 자동 반전.
- **웨이포인트 효과**(★ 4지점 — 출발 t=0·⅓·⅔·도착 t=1, 점 클릭 패널, **외부 클릭 시 닫힘**, 효과 걸린 점엔 점선 링 배지): 구간 속도(0.2~2×, 근처에서 점차 적용·회복) · 점프(포물선 240px) · 말풍선 텍스트.
- 도착 시 충돌 카드 바운스 + 하트 파티클. 발표 중에는 경로선 옅게·컨트롤 숨김(하단 중앙 고정 바로 대체), 유휴 2초 시 자동 숨김(100px 이상 이동 시 재표시).
- GLB 3D 뷰어 연결 시 카메라 헤딩 회전·점프·말풍선 연동. **모션 연결로 뷰어가 프레젠테이션 상태여도 편집 중이면 뷰어 이동·풀스크린 컨트롤은 그대로 노출**(실제 수업 발표 중에만 숨김).
- **양방향 카드 연결(2026-06)**: 요소→모션 지점(a방향)뿐 아니라 **중간 조절점(⅓·⅔=`aMid1`/`aMid2`)도 카드 위로 드래그해 연결**(b방향) — 연결 시 점 둘레에 링 표시, 연결된 점에서 다시 끌어내면 분리. 슬롯 부착은 `attachMotionSlotCmd(node.id, slot, target.id)`(1 undo). 출발/도착(`aStart`/`aEnd`)과 합쳐 4슬롯 모두 양방향.
- **`follow` 유휴 거동**: 연결 요소에 idle='follow'를 주면 무버를 약 420ms 지연으로 뒤따르는 트레일을 그린다(히스토리 버퍼 기반) — "둘이 함께 가는" 연출.

### 16.10 미디어 뷰어 — ★ 매직 뷰어(통합) & 유튜브 추천
- **★ 매직 뷰어(`magic-viewer.html`)**: 유튜브·동영상·3D를 **하나로 다루는 단일 뷰어**(향후 인터랙티브 콘텐츠는 모드 1개 추가로 수용). 담는 내용을 인식해 변신:
  - 유튜브 링크/ID → 유튜브 모드 · 동영상 링크/파일(mp4·webm) → 동영상 모드(재생·탐색·음소거·반복) · GLB/glTF 링크/파일 → 3D 모드(애니메이션 칩). 입력·붙여넣기·**드래그앤드롭**·`?src=` 딥링크로 진입. 무거운 `model-viewer`(three.js)는 **3D를 실제 담을 때만 동적 로드**.
  - **프롬프트 연동**(뷰어 선택 후): 링크/주소 입력 → 그 뷰어에 **즉시 로드**(유튜브·동영상·GLB 자동 판별) / 검색어 → **유튜브 검색**(아래 §유튜브 추천). **동영상 플레이어(`video-player`)는 프롬프트·선 연결로 텍스트/이미지→비디오 생성**(§16.19 — '연결 지점만 마련' 갭 해소).
  - **모드 인지 카드 UI**(`NodeView`): 3D 모드 = 맨몸(투명) + 하단 가로 이동 바 + 모델 클릭 시 전체 UI; 그 외 모드 = 전면 드래그 손잡이 + 우하단 '조작/이동' 토글. **모드·현재 내용(viewerMode/viewerSrc)을 `data`에 영속화 → 새로고침 후 복원**(blob 업로드는 휘발성이라 제외).
- **뷰어 공통 인터랙션 계약**(매직·유튜브·동영상·3D 공유 — `kvSetChrome`/`kvSetPresent`/`loadSrc`/`proposeSearch`/`kv-embed-click`/`kv-embed-drag`/`?fs`):
  - **UI 자동 숨김**: 평소 화면만, **호버/선택 시 입력 줄(크롬) 노출**. 3D는 클릭 시 전체 UI + 커서 이탈 2초 후 페이드.
  - **이동**: 일반 뷰어=화면 전체가 드래그 손잡이(클릭 한 번 선택+드래그). 3D=하단 가로 둥근 바(본문은 카메라 회전이라). 헤더·푸터 **빈 곳 드래그도 카드 이동**(버튼·입력 제외, screen 좌표 전달, 1 undo).
  - **더블클릭=화면 중앙 포커스(2026-06)**: 뷰어 더블클릭은 그 카드를 화면 중앙으로 포커스(`focusNode`) — 이전엔 곧장 조작 모드로 들어가 혼란스러웠다. UI 조작 모드는 **우하단 '조작' 토글**로 진입(Esc 복귀). iframe은 포인터 이벤트를 삼켜 더블클릭이 부모로 전파되지 않으므로, 임베드 문서가 `kv-embed-dblclick` postMessage를 부모에 보내 부모가 **선택+포커스**한다(`public/glb-viewer.html` 등 각 뷰어 HTML에 dblclick 리스너 추가).
  - **풀스크린**: 우상단 ⛶ → **body 레벨 포털 오버레이**(캔버스 변형을 벗어나 화면 전체, 다른 요소 가림·클릭 차단). `?fs` 모드에서 헤더·푸터·종료 ✕는 **마우스 1.5초 무동작 시 페이드아웃**, 움직이면 페이드인. Esc·✕(`kv-fs-exit`)로 닫음. (혼란 주던 per-engine 전체화면 버튼은 제거 — 보드 오버레이로 일원화.)
  - **동영상 플레이어 조작**(§16.19): 드래그 레이어 위(z-30) **재생/정지 버튼**으로 이동과 재생을 양립(버튼만 조작, 나머지 영역 드래그). 헤더·하단 컨트롤 바는 **absolute 오버레이**라 호버 시 나타나도 영상이 밀려 튀지 않고, 컨트롤 바는 호버/선택 시에만 노출(호버 중 상단 52px·하단 56px를 비워 ⛶·슬라이더·음소거·반복 조작 가능).
  - **삭제**: 임베드 iframe이 키 포커스를 가져가도, 선택된 뷰어의 iframe 문서에 단 리스너가 Delete/Backspace를 보드 삭제로 전달.
- **유튜브 추천**: 뷰어에 자료를 선으로 연결하면 자동 추천 —
  - **계획안 문서(또는 그 문서를 담은 프레임)** → 문서 분석(구조화 payload 우선, 마크다운 표 '활동' 열 파싱 폴백)해 **놀이 활동마다 영상 1개씩** 검색, 캡션에 요일 라벨 `(월) …`.
  - 일반 자료 → 유아교육 도메인·교사 니즈 최적화 쿼리(저가 LLM 1콜, 휴리스틱 폴백)로 **3개** 추천.
  - 썸네일(168×94)은 뷰어 아래 가로 중앙으로 로딩 카드부터 즉시 배치, 표시 즉시 **뷰어+썸네일을 '동영상 모음' 프레임으로 감싸기**. ▶ 클릭 시 뷰어에서 재생.
  - 검색 백엔드: `GET /api/youtube/search` — **YouTube API 키 불요**(서버사이드 파싱, 키 비노출).
- **개별 뷰어 프리셋 유지**: 유튜브·동영상 플레이어·GLB 3D 뷰어 프리셋도 그대로(매직 뷰어가 대표). GLB 뷰어는 postMessage 제어·모션 패스 연동·보관함(IndexedDB) 썸네일·모델 다운로드.

### 16.19 ★ AI 동영상 생성 — 텍스트/이미지→비디오 (Gemini Veo)
§16.10 동영상 뷰어를 산출 대상으로 하는 **전용 영상 생성**(이전 §16.18 '프로바이더 미연동' 갭 해소, 2026-06 추가). 교사가 (a) 텍스트 프롬프트, (b) 이미지 카드, (c) 놀이계획/텍스트 카드에서 짧은 **활동 개념 영상**을 만들어 동영상 뷰어에서 바로 재생한다.
- **백엔드(`server/gateway/video.ts`)**: Google **Veo**. 장시간 비동기(11초~6분)라 **2단계** — ① `POST /api/ai/video/start`(`:predictLongRunning`) → 오퍼레이션 이름 ② `GET /api/ai/video/poll?op=…`(오퍼레이션 폴링, `done`이면 **서버가** 결과 mp4 URI를 받아 base64 data URI로 변환). Veo 결과 URI는 키가 있어야 받을 수 있고 **2일 후 만료** → 서버가 즉시 받아 둔 로컬 복사본이 원본이 된다(키 비노출 원칙 유지). 키 없으면 모킹(안내 토스트, 앱 정상).
- **모델·비용**: 기본 `veo-3.0-generate-001`(≈$0.40/초), `KV_GEMINI_VIDEO_MODEL` env로 `veo-3.0-fast-generate-001`(≈$0.15/초)·`veo-2.0-generate-001` 교체. 기본 **4초·16:9·720p**(비용 통제).
- **프롬프트(`buildVeoPrompt`)**: 한국어 요청(또는 계획에서 추출한 활동)을 **공식 5요소**(subject·action·scene/context·camera·composition·ambiance) 영문 Veo 프롬프트로 변환(저티어 LLM 1콜, 실패 시 휴리스틱 폴백). `KV_VIDEO_STYLE`(따뜻한 그림책 애니·잔잔한 카메라·무자막)·`KV_VIDEO_NEGATIVE` 주입.
- **트리거 3종 → 과금·시간 확인 팝오버로 수렴**(`videoGen`): ① **동영상 뷰어 선택 + 프롬프트** = 텍스트→비디오 ② **카드 선택 + "영상 만들어줘"**(`VIDEO_RE` — 검색·추천 어휘는 제외해 유튜브와 충돌 방지) = 카드 옆에 뷰어 스폰 후 생성(이미지 카드=이미지→비디오 / 계획·텍스트 카드=활동 내용 텍스트→비디오, 활동 추출은 유튜브 추천의 `planActivities`/프레임→계획문서 탐색 재사용) ③ **선 연결**(카드 ↔ 동영상 뷰어). 이미지→비디오는 연결 이미지가 **첫 프레임**(`instances[0].image.bytesBase64Encoded`).
- **★ 프롬프트 추가하기**: 확인 팝오버의 생성 버튼에 **호버하면 '프롬프트 추가하기'**가 그 아래에 나타나고(이미지·계획 모드), 클릭 시 **프롬프트 바가 작성 모드**로 전환 — 입력창 placeholder=추천 프롬프트, **연결한 이미지 썸네일이 바 바로 위 가로 중앙**에 뜬다(✕ 취소). 입력(또는 비우면 추천 프롬프트) + 이미지로 생성. 공유 상태 `uiStore.videoCompose`.
- **비용/안전 게이트**: 모든 트리거는 확인 팝오버를 거친다("수십 초~수 분·과금"). `generateVideoForViewer`는 **viewerId 단위 in-flight 가드**로 중복 트리거의 이중 과금을 막고, 공유 정지 시그널(`genSignal`)로 폴링을 끊는다(서버 생성은 계속될 수 있어 게이트로 오발 방지). **사람(아동) 미생성**: 무인물 프롬프트 스타일 + `KV_VIDEO_NEGATIVE`('human faces, real children')로 보장 — ※ Veo 3 텍스트→비디오는 `personGeneration`에서 `allow_all`만 허용(실측 HTTP 400으로 `dont_allow`·`allow_adult` 거부 확인)이라 값 미전송(기본)+프롬프트로 통제.
- **영속화·복원**: 생성 mp4(data URI)는 **`video-asset:v1:<id>` IndexedDB**에 id별 키로 보관하고 스냅샷엔 `node.data.videoAssetId`만 둔다(대용량 data URI를 `viewerSrc`에 넣지 않아 스냅샷 비대화 방지). 새로고침 시 `videoAssetId`로 IDB에서 받아 뷰어에 `loadSrc` 복원.
- **동영상 뷰어 조작**(`video-player.html`·`NodeView`): ① 드래그 레이어 위(z-30) **재생/정지 버튼** — 이동과 재생 양립(버튼만 조작, 나머지 드래그). iframe `kvTogglePlay` 프록시 + `kv-video-playing` 상태로 아이콘 동기화. ② 헤더·하단 컨트롤 바를 **absolute 오버레이**로 전환 → 호버 시 나타나도 영상이 밀려 튀지 않고(레이아웃 고정), 컨트롤 바는 **호버/선택 시에만** 표시. 호버 중 드래그 레이어가 상단 52px·하단 56px를 비워 헤더(⛶)·슬라이더·음소거·반복을 바로 조작.
- **거버넌스**: 정책 `video` 추가 — 생성=L1·과금 확인 게이트 / 공유=L2 / 아동 미생성. EvalPage 체크리스트에 노출.

### 16.22 ★ 인터렉티브 노드 — 보드 네이티브 인터랙티브 카드 (2026-06-21 신규)
보드 툴바에서 만드는 **네이티브 보드 노드 타입(`type:'interactive'`)** — 게임뷰어(§16.21, iframe·GameSpec)와는 별개로, **보드 카드 자체가 파스텔 인터랙티브 캔버스**다. 교사가 사진·글자·도형을 올리고 "탭하면 일어나는 일"을 붙여 아이용 놀이 카드를 만든다. 코드: `src/features/interactive-viewer/**`. 영속화 localStorage `kv:inodes:v1`, 보드 노드는 `data.docId`만 참조(슬라이드 `?id=` 선례). 풀스크린 저작/재생은 `ZoomOverlay` 재사용.

- **단일 계약 스키마**(`schema/interactiveNode.ts`, zod, **재정의 금지** — 사전검증 산출물): elements(image/video/text/shape/sprite)·connections(path/link/order)·behaviors(트리거→액션)·conditions(when)·counters/flags·story(graph)·groups. 참조 무결성 superRefine. 생성·저작·런타임·수업모드가 모두 이 문서만 의존.
- **동작 엔진**(`runtime/InteractiveStage.tsx` `fireBehavior`): 액션 **11종** 실행 — 반응(WAAPI 9프리셋)·교체·보이기/숨기기·강조·세기·스위치(플래그)·말하기(말풍선+브라우저 TTS ko-KR)·영상재생·연결 따라 이동(+장면전환은 no-op). 제어: **조건(when, 카운터/플래그/상태)·지연(delay)·잇기(then 체이닝)**. 카운터 배지·말풍선·숨김/강조 상태를 렌더, 리셋 토큰으로 지연/체인 취소. 🐛 영상 자동재생 금지(playVideo/컨트롤로만).
- **트리거 6종**: 탭하면(tap)·시작하면(sceneEnter, 재생 시 자동)·순서대로 탭(sequenceTap, 연결 순번대로만 발화·틀리면 흔들)·끌어서 잇기(pathTraverse, 재생서 연결 대상 위로 드래그)·이야기 넘길 때(storyAdvance)·완료후(afterComplete).
- **연결**(마이보드 포트 시스템 일부 재사용 `@/board/links`): 요소 호버 → 좌/우 포트 드래그로 from→to, **슬롯 스택(한 요소 다중 연결)·떼어내기(빈 곳=해제)·순번 라벨**(가지=‘다음 단계 번호’ 기준, 1→2/2-1, 2→3/3-1). 연결이 sequenceTap/pathTraverse/moveAlongPath의 입력이 된다.
- **이야기(story)**: 단계별 나레이션 재생기(하단 자막 바 + TTS, ◀/다음/완료) + 저작 패널 `authoring/StoryPanel`(단계 추가·순서·삭제). 분기(branches)는 런타임 지원(조건 맞는 step으로 점프), 분기 전용 저작 UI는 후속.
- **저작 UX(마이보드 상속)**: 이동·리사이즈(비-멤버 등비, 회전 시 중심 기준 보정)·회전 핸들·다중선택·단축키·실행취소, **글자 더블클릭 인라인 편집(박스맞춤 반응형 FitText)**, **이미지 호버 액션(편집/다운로드/풀스크린 — 마이보드 `ImageEditorModal`·`ImageFullscreen` 재사용)**, **🧩 묶어서 복제**(선택 다요소를 동작·내부연결까지 id 재매핑해 통째 복제 — 프리팹). 인스펙터에서 동작·트리거·실행 조건 저작.
